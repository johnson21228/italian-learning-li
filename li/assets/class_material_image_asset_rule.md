# Class Material Image Asset Rule

Italian class images are learning assets only when they stay attached to a real class phrase, situation, and speaking loop.

## Rule

Images that correspond with class material should be captured as **class material image assets**. They are not decorative extras. They should help the learner notice, name, ask, answer, repair, and speak in Italian.

A class material image asset must include:

- source image path
- class phrase or phrase family
- situation where the phrase is useful
- first-person learner line
- call-and-response practice
- pronunciation note
- repair phrase
- personal corpus capture fields

## Standard location

Use this repository location for image assets derived from class material:

```text
assets/class_material/
```

For first-class Italian material, use:

```text
assets/class_material/first_class/
```

## Image-to-corpus posture

The image is not the lesson by itself. The image becomes useful when it supports a small exchange such as:

```text
A: Come ti chiami?
B: Mi chiamo Steve.

A: Come si chiama questo?
B: Si chiama libro.

A: Non capisco.
B: Puoi ripetere lentamente?
```

## Grammar position

Do not begin by explaining the grammar of `chiamarsi`. Begin by using the phrase in speech. Grammar may appear later as clarification:

```text
Mi chiamo = I call myself / my name is
Si chiama = it is called / his or her name is
```

## Required loop

Each class image should be usable in the existing loop:

```text
Hear → Imitate → Answer → Repair → Capture
```

## Anti-pattern

Do not create generic language-learning posters detached from the learner's class material. The image should correspond to actual class phrases, slides, notes, or practice needs.
